# 注意事项
## 指令发出的方式有两种
1. 通过 send_msg 轮询接口发出
2. 通过 receive_msg 接口 return 发出

## 接口所用编码均为utf-8编码

## 服务端发出的指令格式应该是一个json数组，示例：
```json
[{"sendId":"","api":"sendTextMessage","option":{"wxid":"","text":""}}]
```

# receive_msg
## 参数说明
|数据格式中的参数|参数的含义|
|:--------------|:-------|
|action         |上报的名称|
|cwxid          |当前登录微信账号的微信 ID|
|data           |上报的数据|

## 数据格式
```json
{
    "action" : "",
    "cwxid" : "",
    "data" : {}
}
```

### send_msg 任务下发到DLL的响应
```json
{
    "action":"",
    "cwxid":"",
    "data":{"errorReason":"","sendId":"","sendResult":""},
}
```
